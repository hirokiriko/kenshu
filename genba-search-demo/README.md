# GENBA Search デモ（PoC / MVP）— 無効資料調査支援版

無効資料調査において、対象特許（イ号）クレームの**正式用語で検索しても漏れてしまう**、
現場の方言で書かれた公知資料を拾い上げ、該当箇所をハイライトするデモです。
すべてブラウザ内のダミーデータで動作し、サーバー・APIは不要です（GitHub Pages でそのまま動きます）。

## 使い方の流れ（前バージョンとは逆向き）

1. 対象特許（イ号）クレームの文言を入力（例：「周期的な相対変位…縞状の模様…」）
2. 通常キーワード検索の結果（件数が多く埋もれやすい）を表示
3. **キーワードを含まないのに同概念を方言で記した公知資料**を追加抽出し、
   該当箇所をハイライト＋「クレーム用語 ⟷ 方言表現」の対応表＋判断根拠を表示
4. 「方言箇所をハイライト表示」チェックで、強調の有無を切り替え可能

## 収録クレーム例

- 例1：周期的な相対変位／縞状の模様 → 「ビビリ」「バイトが鳴く」「波目」等を拾う
- 例2：凝着／難削材 → 「焼き付き」「刃に材料が持っていかれる」「構成刃先」等を拾う

## GitHub Pages での公開手順

1. GitHub で新しいリポジトリを作成（例: `genba-search-demo`、Public）
2. この `index.html` をリポジトリ直下に置いて push
3. リポジトリの **Settings → Pages** を開く
4. Source: **Deploy from a branch** / Branch: **main** / フォルダ: **/(root)** を選択して Save
5. 数分後、`https://<ユーザー名>.github.io/genba-search-demo/` で公開されます

## 注意

公報番号・出願人・本文・ヒット件数は、すべて動作イメージ確認用の架空ダミーデータです。
実在の特許公報ではなく、検索処理もブラウザ内のサンプルで模擬しています。
